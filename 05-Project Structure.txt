//! Installing React.js using Vite
--> Vite is a modern build tool used to create fast React applications.

//! Prerequisites
--> Install Node.js before creating a React project.
--> Node.js includes npm (Node Package Manager).

Check Installation:
node -v
npm -v
--> If version numbers are displayed, Node.js is installed successfully.

//! Step 1: Create a React Project
Command: npm create vite@latest
After running the command:
✔ Project name: my-react-app
✔ Select a framework: React
✔ Select a variant: JavaScript

//! Step 2: Move into Project Folder
Command: cd my-react-app

//! Step 3: Install Dependencies
Command: npm install or npm i
--> Downloads all required packages mentioned in package.json.
--> Creates the node_modules folder.

//! Step 4: Start Development Server
Command: npm run dev

//! React Project Structure
--> A React project contains multiple files and folders that help organize the application.
--> Each folder has a specific purpose.
--> Proper project structure makes applications easy to maintain and scale.

//! React Project Structure
my-react-app/
│
├── node_modules/
├── public/
├── src/
├── .gitignore
├── package.json
├── package-lock.json
├── README.md

//! 1. node_modules
--> Contains all installed dependencies and packages.
--> Created automatically after running npm install.
--> Usually the largest folder in a React project.
--> Should not be modified manually.
//*Example:
react
react-dom
vite
axios
react-router-dom

//*Note:
--> This folder is ignored by Git using .gitignore.

//! 2. public
--> Contains static files that do not go through React processing.
--> Files inside public can be accessed directly by the browser.
//*Common Files:
public/
│
├── favicon.ico
├── images/
└── index.html

//*Note:
--> Used for static assets like images, icons, and PDFs.

//! 3. src/
--> Most important folder in a React project.
--> Contains all React code.
--> Components, CSS, Hooks, Pages, and Logic are usually placed here.

//*Example:
src/
│
├── components/
├── pages/
├── assets/
├── App.jsx
├── main.jsx
└── index.css

//! 4. App.jsx
--> Root Component of the application.
--> Acts as the main UI container.
--> Other components are usually rendered inside App.jsx.

//! 5. main.jsx
--> Entry Point of the React application.
--> Responsible for rendering App Component into the browser.

//*Note:
--> Execution starts from main.jsx.

//!6. package.json
--> Heart of the React project.
--> Contains project information and dependencies.

Stores:
--> Project name
--> Version
--> Scripts
--> Installed packages

//*Example:
{
  "name": "my-app",
  "version": "1.0.0"
}

//! 7. package-lock.json
--> Automatically generated by npm.
--> Stores exact dependency versions.
--> Ensures consistent installation across systems.

//! 8. .gitignore

--> Specifies files and folders Git should ignore.

Common Entries:

node_modules
dist

Advantage:
--> Prevents unnecessary files from being uploaded to GitHub.

//! 9. README.md
--> Documentation file for the project.
--> Contains project description, setup steps, and usage instructions.
