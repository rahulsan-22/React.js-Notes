//! Props

//! What are Props?
--> Props stands for Properties.
--> Props are used to pass data from Parent Component to Child Component.
--> Props always returns an object
--> Props make Components reusable and dynamic.
--> Props are Read-Only (Immutable).

//!Simple Definition:
--> Props are inputs to a Component.

//! Example Without Props
function Welcome() {
    return <h1>Hello Yasin</h1>;
}
--> Fixed data.
--> Cannot be reused for different users.

//! Example With Props
function App() {
    return <>
    <Welcome name="Pavan" />
    <Welcome name="Monty" />
    </>;
}

function Welcome(props) {
    return <h1>Hello {props.name}</h1>;
}

//*Output:
Hello Pavan
Hello Monty

--> Dynamic data using Props.

//! How Props Work?
Parent Component
       ↓
    Props
       ↓
Child Component

//*Example:
function App() {
    return <Child name="Rahul" />;
}

function Child(props) {
    return <h1>{props.name}</h1>;
}

//! Passing Multiple Props
const Nanna = () => {
  return <>
    <Koduku item="Car" price={100000} worthy={true} />
  </>
}

const Koduku = (props) => {
  return (
    <div>
      <h1>Koduku received {props.item} from his Nanna which is worth of ₹{props.price}</h1>
      <p>Is he worth of {props.item}: {props.worthy ? "true":"false"}</p>
    </div>
  )
}

//*Output:
Koduku received Car from his Nanna which is worth of ₹100000
Is he worth of Car: true

//! Props are Read-Only
function App(){
    return <Student name="Rahul"/>
}
function Student(props) {
    props.name = "Kiran";   ❌ Error
    return <h1>{props.name}</h1>;
}

Reason:
--> Props cannot be modified by Child Components.
--> Data flows only from Parent to Child.

//! Destructuring Props

//*Without Destructuring:
function Student(props) {
    return <h1>{props.name}</h1>;
}

//*With Destructuring:
function Student({ name }) {
    return <h1>{name}</h1>;
}

//*Multiple Props:
function Student({ name, age }) {
    return (
        <>
            <h1>{name}</h1>
            <h2>{age}</h2>
        </>
    );
}

Advantage:
--> Cleaner and more readable code.

//! Passing Different Data Types
1. String
<Greeting name="Rahul" />

2. Number
<Student age={22} />

3. Boolean
<User isLoggedIn={true} />

4. Array
<List fruits={["Apple","Mango"]} />

5. Object
<User user={{name:"Rahul", age:22}} />

6. Function
<Button handleClick={showMessage} />

//! Passing Function as Props
function App() {
    function showMessage() {
        alert("Hello Student!");
    }
    return (
        <Child handleClick={showMessage} />
    );
}

function Child({ handleClick }) {
    return (
        <button onClick={handleClick}>
            Click
        </button>
    );
}

--> Commonly used for Parent-Child communication.

//! Props with Default Values
function Welcome({ name = "Student" }) {
    return <h1>Hello {name}</h1>;
}

//*Usage:
<Welcome/>

//*Output:
Hello Student

//! Children Prop
--> Every Component automatically receives a special prop called children which is the data between the parent component.
--> Used to display nested content.

//*Example:
function App() {
    return (
        <Card>
            <h1>Hello Student</h1>
        </Card>
    );
}
function Card(props) {
    return (
        <div>
            {props.children}
        </div>
    );
}

//*Output:
Hello Student

//! Prop Drilling
-->Prop Drilling is the process of passing data (props) from a parent component to a deeply nested child component through multiple intermediate components, even when those intermediate components do not need the data.

//*Example:
App
 ↓
Parent
 ↓
Child
 ↓
GrandChild

--> Data is passed through every level.

//*Problem:
--> Code becomes difficult to manage.

//*Solution:
--> Context API
--> Redux etc..
