//! useState() Hook

//! What is useState()?
--> useState() is a built-in Hook introduced in React 16.8.
--> It is used to create and manage State in Functional Components.
--> Whenever the State changes, React automatically re-renders the Component.

//*Definition:
--> useState() is a Hook that allows Functional Components to store and update dynamic data.

//! Syntax
const [state, setState] = useState(initialValue);

//! Explanation
state
--> Stores the current State value.

setState
--> Function used to update the State.

initialValue
--> Initial value assigned to the State.

//! Why useState()?
--> Store dynamic data.
--> Update the UI automatically.
--> Handle user interactions.
--> Replace State used in Class Components.

//! Import
import { useState } from "react";

//! Example 1: Number State
import { useState } from "react";

function Counter() {
    const [count, setCount] = useState(0);
    return (
        <>
            <h1>{count}</h1>
            <button onClick={() => setCount(count + 1)}>
                Increment
            </button>
            <button onClick={() => setCount(count - 1)}>
                Decrement
            </button>
            <button onClick={() => setCount(0)}>
                Reset
            </button>
        </>
    );
}

//! Flow
Initial State
      ↓
User Clicks Button
      ↓
setCount()
      ↓
State Updated
      ↓
Component Re-rendered


//! Example 2: String State
function App() {
    const [name, setName] = useState("Rahul");
    return (
        <>
            <h1>{name}</h1>
            <button onClick={() =>setName("Kiran")}>
                Change Name
            </button>
        </>
    );
}

//! Example 3: Boolean State
function App() {
    const [isLoggedIn, setIsLoggedIn] = useState(false);
    return (
        <>
            <h1>
                {isLoggedIn ? "Welcome" : "Please Login"}
            </h1>
            <button onClick={() =>setIsLoggedIn(!isLoggedIn)}>
                Toggle
            </button>
        </>
    );
}

//! Example 4: Object State
function App() {
    const [student, setStudent] = useState({
        name: "Rahul",
        age: 22
    });
    function updateAge() {
        setStudent({
            ...student,
            age: 23
        });

    }

    return (
        <>
            <h1>{student.name}</h1>
            <h2>{student.age}</h2>

            <button onClick={updateAge}>
                Update Age
            </button>
        </>
    );
}

Reason:
--> Spread Operator preserves existing properties.

//! Example 6: Array State

function App() {

    const [fruits, setFruits] = useState([
        "Apple",
        "Mango"
    ]);

    function addFruit() {

        setFruits([
            ...fruits,
            "Orange"
        ]);

    }

    return (
        <>
            {
                fruits.map((fruit, index) => (
                    <h2 key={index}>
                        {fruit}
                    </h2>
                ))
            }

            <button onClick={addFruit}>
                Add Fruit
            </button>
        </>
    );
}

//! Functional Updates

--> Use when the next State depends on the previous State.

❌ Normal Update

setCount(count + 1);

✔ Functional Update

setCount((prevCount) => prevCount + 1);

Advantage:
--> Always receives the latest State value.

//! Example

function Counter() {

    const [count, setCount] = useState(0);

    function increment() {

        setCount((prevCount) => prevCount + 1);

    }

    return (
        <>
            <h1>{count}</h1>

            <button onClick={increment}>
                Increment
            </button>
        </>
    );
}

//! Multiple State Variables

function App() {

    const [name, setName] = useState("Rahul");
    const [age, setAge] = useState(22);
    const [city, setCity] = useState("Hyderabad");

}

//! Lazy Initialization

--> Used when calculating the initial State is expensive.

Example:

const [count, setCount] = useState(() => {

    console.log("Executed Only Once");

    return 0;

});

//! Rules of useState()

Rule 1:
--> Call useState() only at the Top Level.

✔ Correct:

const [count, setCount] = useState(0);

❌ Wrong:

if (true) {
    useState(0);
}

Rule 2:
--> Call useState() only inside Functional Components or Custom Hooks.

//! Common Mistakes

❌ Mistake 1:

count = count + 1;

Reason:
--> State should never be updated directly.

✔ Correct:

setCount(count + 1);

❌ Mistake 2:

student.age = 23;

Reason:
--> Mutating Object State directly.

✔ Correct:

setStudent({
    ...student,
    age: 23
});

❌ Mistake 3:

fruits.push("Orange");
setFruits(fruits);

Reason:
--> Mutates the existing Array.

✔ Correct:

setFruits([
    ...fruits,
    "Orange"
]);

//! useState() vs State in Class Components

| useState() | Class Component State |
|-------------|-----------------------|
| Functional Components | Class Components |
| useState() Hook | this.state |
| setState Function | this.setState() |
| No this Keyword | Uses this Keyword |
| Simpler Syntax | More Boilerplate |

//! Interview Points

--> useState() was introduced in React 16.8.
--> useState() is used only in Functional Components.
--> useState() returns an array containing the State value and the updater function.
--> Updating State causes the Component to re-render.
--> State should never be modified directly.
--> Functional Updates should be used when the next State depends on the previous State.
--> Spread Operator is commonly used while updating Objects and Arrays.
--> Multiple useState() Hooks can be used in the same Component.
--> Lazy Initialization executes only once during the initial render.
